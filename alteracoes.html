Adicionando alterações

Agora que estamos prontos para codar com nossa pasta na máquina e o repositório no GitHub é hora de programar!

Quando você estiver com a sua parte inicialmente feita e quiser mandar para o repositório iremos utilizar primeiramente a função git add.

Primeiro abra o terminal do git na IDE, no caso do VS Code ele será aberto ao apertar Ctrl + Shift + ". Essa função possui algumas versões. São elas:

  • Para adicionar mudanças em uma página em especifíco: git add nome_do_arquivo
  • Para adicionar as alterações na pasta local: git add.
  • Adicionar todas as mudanças feitas: git add -A
  • Para desfazer o add use o seguinte comando: git reset OU git reset nome_do_arquivo

Após realizar o seu comando add vamos verificar se deu tudo certo. Utilize o comando git status deve aparecer uma mensagem de confirmação de que foi adicionada.

Ao finalizar a adição do seu arquivo iremos para o commit isso irá confirmar o envio das suas mudanças. Utilizamos o código: commit -m "comentário da alteração" é importante você adicionar um comentário coerente com o que foi feito para ajudar o seu time no processo de construção do programa.

Feito tudo isso vamos enviar para o repostiório remoto. Para isso vamos usar o comando: git push ele irá ficar assim mesmo, sem o link do repositório ou nome do arquivo. Pois irá enviar tudo que foi comitado por você.

Agora que você já sabe como adicionar as suas alterações ao repositório do GitHub vamos ver como obter as alterações feitas pelos seus colegas de time.

Para verificarmos se há novas alterações no repostirório utilizamos da função git fetch ele irá irformar no terminal se foram feitas atualizações no repostiório por outras pessoas.

Agora para você adicionar ass alterações feitas pelos outros na sua máquina vamos usar do comando git pull assim tudo que foi atualizado no repositório será salvo na sua máquina.
