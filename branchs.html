Trabalhando com branchs

As branchs servem para que durante o desenvolvimento do programa possam ser desenvolvidas funcionalidades extras que serão incorporadas a master(linha principal de códigos do programa). Toda vez que você cria um repostiório é criada uma branch, a main, depois disso vamos criar uma nova branch. Temos algumas opções para realizar essa tarefa.

Para criarmos uma nova branch podemos usar os seguintes comandos:

  • git branch nome_da_branch --- esse comando apenas cria a nova branch;
  • Mesmo criando uma nova branch é necessários irmos para ela, pois por padrão, e utilizando o comando acima não vamos diretamente para a nova branch. Para trocar de branch usamos: git checkout nome_da_branch .
  • Mas para nos poupar de tanto trabalho podemos usar: git checkout -b nome_da_branch --- esse comando irá automáticamente criar uma nova branch e já mudar sua localização para essa nova branch.
  • Caso você queira excluir a branch que está usa-se esse comando: git branch -d nome_da_branch .
  • Agora o item mais importante de todos. Para que os outros participantes do seu time tenham acesso a essa branch que você criou é necessário usar o seguinte comando: git push --set-upstream origin nome_da_branch.
